EA1. Proyecto integrador
Ingestión de Datos desde un API
 

Objetivo
Implementar la etapa de ingesta de datos del proyecto integrador de Big Data. En esta actividad se debe:

Leer datos desde un API.

Almacenar los datos en una base de datos (recomendación  SQLite).

Generar evidencias complementarias:

Un archivo generado con Pandas (Excel o CSV) que represente una muestra de los registros en la base de datos.

Un archivo de auditoría en formato .txt que compare los registros extraídos del API con los almacenados en la base de datos.

Alojar el proyecto en un repositorio de GitHub y configurar GitHub Actions para la ejecución automatizada, dejando evidencia de la creación de la base de datos y de los archivos generados.

Esta actividad es crítica, ya que las siguientes etapas del proyecto (preprocesamiento, enriquecimiento y modelado) dependerán de la correcta ingesta y estructuración de los datos.

 

Instrucciones
Lectura de Datos desde un API:
Desarrolla un script en Python que se conecte a un API y extraiga los datos requeridos.

Almacenamiento en SQLite:
Utiliza el módulo de base de datos (sugerencia sqlite3 ) para crear una base de datos analítica.

Diseña y crea las tablas necesarias en la base de datos.

Inserta los datos extraídos del API en la base de datos.

Generación de Evidencias Complementarias:
Archivo de Muestra con Pandas:

Emplea la librería Pandas para procesar los datos y generar un archivo en formato Excel o CSV que contenga una muestra representativa de los registros.

Archivo de Auditoría (.txt):

Crea un script que compare los registros extraídos del API con los registros almacenados en la base de datos.

Guarda el resultado de la comparación en un archivo de texto (.txt).

Alojamiento y Automatización:
Aloja todo el proyecto (código fuente, scripts, base de datos y archivos generados) en un repositorio de GitHub.

Configura GitHub Actions para ejecutar automáticamente el script de ingesta. El workflow debe:

Ejecutar la lectura del API.

Actualizar la base de datos SQLite.

Generar el archivo de muestra y el archivo de auditoría.

Dejar evidencia en el repositorio (por ejemplo, subiendo los archivos generados como artefactos o dejándolos en una carpeta de salida).

 

Desarrollo o ruta metodológica
Extracción de Datos desde el API:
Selección y conexión:

Identifica el API a utilizar y revisa su documentación para conocer los endpoints y parámetros necesarios.

Desarrolla un script en Python (utilizando librerías como requests) que se conecte al API y realice la solicitud de datos.

Extracción:

Ejecuta el script para extraer los datos y almacena la respuesta (por ejemplo, en formato JSON).

Almacenamiento en una Base de Datos SQLite:
Creación de la base de datos:

Utiliza el módulo sqlite3 de Python para crear una base de datos analítica.

Diseño y creación del esquema:

Define las tablas y campos necesarios para almacenar la información extraída.

Ejecuta las sentencias SQL correspondientes para crear la estructura de la base de datos.

Inserción de datos:

Procesa la respuesta del API y, mediante el script, inserta los registros en las tablas de la base de datos.

Generación de Evidencias Complementarias:
Archivo de muestra con Pandas:

Carga los datos extraídos en un DataFrame utilizando Pandas.

Selecciona una muestra representativa de los registros (por ejemplo, las primeras N filas) y expórtalos a un archivo en formato Excel o CSV.

Archivo de auditoría (.txt):

Lee los datos almacenados en la base de datos SQLite (por ejemplo, con pandas.read_sql_query).

Compara estos registros con los datos obtenidos directamente del API.

Genera un reporte de auditoría en un archivo de texto que detalle, por ejemplo, el número de registros extraídos, posibles diferencias en campos clave y confirmación de la integridad de los datos.

Alojamiento y Automatización en GitHub:
Repositorio:

Crea un repositorio en GitHub y sube todo el proyecto (código fuente, scripts, base de datos generada y archivos de evidencia).

Automatización con GitHub Actions:

Configura un workflow (archivo YAML en .github/workflows) que defina un job para:

Instalar Python y las dependencias necesarias (como requests, pandas, etc.).

Ejecutar el script de ingesta de datos.

Verificar la creación de la base de datos y la generación de los archivos (muestra y auditoría).

(Opcional) Publicar o almacenar los archivos generados como artefactos del workflow para facilitar su revisión.

Verificación y Validación:

Realiza pruebas locales del script para asegurarte de que:

Los datos se extraen correctamente desde el API.

La base de datos SQLite se crea y se llena con la información esperada.

Los archivos de evidencia (Excel/CSV y .txt de auditoría) contienen los datos correctos.

Ejecuta el workflow de GitHub Actions y revisa los logs y artefactos generados para confirmar que todo el proceso se ejecuta sin errores.
